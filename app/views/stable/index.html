Prototype 1.2 April 2017 to present

Summary of changes

Sprint 3-4

  • Complete content review with Greg Tindall (previously no dedicated resource)
  • Inline error messaging for files of the wrong format or too large
  • New sections created to update your organisation's email address
  • Changed functionality of upload area so that files do not automatically upload as per the previous version. Due to the size of the files we wanted to give the user control to select the files first, before confirming their upload.
  • Added a section for user registration with the service. This is actually being built by another team however we wanted to user test as part of the end to end journey.

Changes from previous user testing (sprints 1-2)

  • Terms and conditions declaration statement with download option
  • Update to displaying the entirety of the file name which previouly used HTML ellipsis, cutting off most of the characters. User testing showed that users are familiar with the format of the file and can instantly recognise what the file contains, if they can see the whole thing.
